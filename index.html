    Showcase Template #

    Welcome to the gohugo template to create rich content academic reports having p5.js sketches.

    Hacking #

    Create a github user account or organization, and install git and the gohugo static site generator then:

    1. Login into GitHUb and create a new repo from this template into your user account or organization. Don’t rename the repo but leave it as ‘showcase’.
    2. Grant read and write permissions to your newly created repo workflow. Observation: If you’re deploying an organization site this permission need to be granted within the organization settings.
    3. Clone your newly created repo.
    4. Install the site default theme:
      $cd showcase
$git submodule update --init --recursive
    5. Redefined baseURL in hugo.toml to point to your site url, i.e., https://username.github.io/showcase/ where username is your username (or organization name) on GitHub. Refer to GitHub pages for details about url naming conventions. Don’t forget to commit your changes.

    Render your site locally: $hugo server -D --disableFastRender.

    Deploy with $git push. Don’t forget to select the gh-pages branch as the one to serve your site from at the pages section of your repo configuration page.

    Remark #

    The showcase template uses the hugo-book theme by default. Check the hugo themes site if you wish to add other ones.
